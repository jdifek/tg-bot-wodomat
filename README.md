# 🤖 Watermat Monitor Bot

Telegram-бот для мониторинга водоматов через API Shengtiqi (生钛圈).  
Поддерживает **русский** и **польский** языки.

---

## 📋 Что отслеживает бот

| # | Событие | Как детектирует |
|---|---------|-----------------|
| 1 | 🚱 Нет воды в баке | API `exception-status-query` → waterLevel |
| 2 | 💸 Продажа без розлива | `record/getlist` → оплата > 0, вода = 0 |
| 3 | 🥶/🔥 Температура вне нормы | `device/getdetail` → поле temp |
| 4 | 📵 Аппарат оффлайн 15+ мин | `exception-status-query` → lastConnect |
| 5 | 📊 Ежедневный отчёт продаж | `record/getlist` за прошедший день |
| 6 | 🔴 Статус онлайн/оффлайн | `exception-status-query` → statusMsg |
| 7 | 💳 Удалённое пополнение QR | `addvalue/getlist` — новые записи |
| 8 | 📶 Истекает SIM-карта | `simcard/getlist` → valid_date |

---

## ⚙️ Требования

- **Node.js** версии 18 и выше
- Доступ к интернету
- Telegram Bot Token (получить у [@BotFather](https://t.me/BotFather))
- Данные API Shengtiqi: `appid` и `saler`

## 🚀 Установка на сервер

### 1. Распакуйте архив

```bash
unzip watermat-monitor-bot.zip
cd watermat-monitor-bot
```

### 2. Установите зависимости

```bash
npm install
```

### 3. Настройте конфигурацию

```bash
cp .env.example .env
nano .env
```

Заполните обязательные поля:

```env
# Токен бота от @BotFather
BOT_TOKEN=1234567890:AABBCCDDEEFFaabbccddeeff

# Язык: ru (русский) или pl (польский)
LOCALE=ru
```

> ⚠️ `APPID` и `SALER` **НЕ задаются в .env** — они передаются через ссылку при запуске бота каждым пользователем.

### 4. Запустите бота

```bash
# Обычный запуск
npm start

# Для постоянной работы (рекомендуется PM2)
npm install -g pm2
pm2 start npm --name "waterbot" -- start
pm2 save
pm2 startup
```

## 🔗 Как пользователь подключается к боту

В вашей админ-панели сформируйте ссылку:

```
https://t.me/<ИМЯ_БОТА>?start=APPID_<appid>_SALER_<saler>
```

**Пример:**
```
https://t.me/MyWaterBot?start=APPID_2beea5cb3ce00296dba32e36b8a9dc3f_SALER_myshop123
```

Пользователь нажимает на ссылку → открывается бот → нажимает `/start` → бот автоматически подключается к его аппаратам.

## ⏱ Логика опроса (для 200 аппаратов)

```
Каждую минуту:
├── Быстрая проверка аномалий (1 запрос на тип устройства)
├── Детальный опрос батча 90 аппаратов (температура)
├── Проверка QR платежей (последние 15 мин)
└── Проверка SIM карт (раз в час)

Каждые 5 минут:
└── Отправка накопленных алертов в Telegram

Каждый день в 08:00:
└── Отчёт продаж за прошедший день
```

**Батчинг:** 200 аппаратов разбиваются на батчи по 90.  
За 3 минуты все 200 аппаратов опрошены полностью.  
Лимит API (100 запросов/мин) не превышается.

## 🔧 Параметры .env

| Параметр | По умолчанию | Описание |
|----------|-------------|----------|
| `BOT_TOKEN` | — | **Обязательно.** Токен бота |
| `LOCALE` | `ru` | Язык: `ru` или `pl` |
| `API_BASE_URL` | `http://api.happy-ti.com:2028` | Адрес API |
| `POLL_INTERVAL_SEC` | `60` | Интервал опроса API (сек) |
| `ALERT_INTERVAL_SEC` | `300` | Интервал отправки алертов (сек) |
| `BATCH_SIZE` | `90` | Аппаратов за один батч |
| `REQUEST_DELAY_MS` | `350` | Задержка между запросами (мс) |
| `TEMP_MIN` | `3` | Минимальная температура (°C) |
| `TEMP_MAX` | `40` | Максимальная температура (°C) |
| `OFFLINE_MINUTES` | `15` | Порог оффлайна (мин) |
| `SIM_DAYS_WARN` | `7` | За сколько дней предупреждать об истечении SIM |
| `DAILY_REPORT_HOUR` | `8` | Час отправки ежедневного отчёта |
| `DEVICE_TYPES` | `shop,shop_liquid,shop_water` | Типы устройств для мониторинга |

## ❓ Частые вопросы

**Q: Бот забывает пользователей после перезапуска?**  
A: Да, сейчас состояние в памяти. После перезапуска пользователи заново нажимают `/start` через ту же ссылку.

**Q: Можно ли добавить несколько пользователей?**  
A: Да, каждый пользователь подключается через свою ссылку с `appid` и `saler`. Бот одновременно мониторит всех.

**Q: Лимит 100 запросов/мин — что если аппаратов больше 200?**  
A: Увеличьте `POLL_INTERVAL_SEC=120` и уменьшите `BATCH_SIZE=80`. Батч будет опрашиваться за 2 минуты вместо 1.

**Q: Как поменять язык на польский?**  
A: В `.env` установите `LOCALE=pl` и перезапустите бота.
